## Purpose

This section defines **mode authority** for creative work in this IP.

The problem is not switching cognitive modes.  
The problem is **allowing more than one mode to hold authority at the same time**.

Mode overlap produces:
- incoherent decisions
- scale drift
- revision paralysis
- rule-of-cool violations
- erosion of invariants

This system exists to prevent that failure.

---

## Core Rule (Non-Negotiable)

> **Only one mode has authority at any given time.**

All other modes are silent observers.

If authority is unclear, **no work is valid** until authority is declared.

## Conflict Resolution Rule

When modes disagree:

- Muse defers to Architect
- Architect defers to Invariants
- Editor defers to Architect
- Operator defers to Schedule, not preference

No mode overrules the invariants.

## Relationship to Narrative Invariants

- **Muse Mode** is allowed to violate invariants temporarily.
- **Architect Mode** exists to enforce invariants absolutely.
- **Editor Mode** clarifies invariant expression.
- **Operator Mode** locks invariant-compliant outcomes into canon.

If invariants are violated in final output, mode authority failed.
